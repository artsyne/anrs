## рҹ“Ұ Components

### 1. Rules Layer (`ai/rules/`)

**Purpose**: Define constraints and standards.

```
ai/rules/
в”ңв”Җв”Җ global.md        # Critical constraints
в”ңв”Җв”Җ coding.md        # Code standards
в”ңв”Җв”Җ safety.md        # Security rules
в””в”Җв”Җ constraints.json # Machine-readable rules
```

### 2. State Layer (`ai/state/`)

**Purpose**: Track execution state (SSOT).

```
ai/state/
в”ңв”Җв”Җ state.json       # Current state
в”ңв”Җв”Җ state.schema.json
в””в”Җв”Җ scratchpad/      # Temporary memory
```

### 3. Skills Layer (`ai/skills/`)

**Purpose**: Define executable actions.

```
ai/skills/
в”ңв”Җв”Җ index.json       # Skill registry
в”ңв”Җв”Җ core/            # Essential skills
в”ңв”Җв”Җ engineering/     # Dev skills
в”ңв”Җв”Җ sre/             # Reliability skills
в””в”Җв”Җ env/             # Environment skills
```

### 4. Orchestrator (`ai/orchestrator/`)

**Purpose**: Control execution flow.

```
ai/orchestrator/
в”ңв”Җв”Җ ORCHESTRATOR.md  # Main protocol
в””в”Җв”Җ strategies/      # Execution strategies
```

### 5. Harness (`harness/`)

**Purpose**: Evaluate code quality.

```
harness/
в”ңв”Җв”Җ quality_gate.py  # Entry point
в”ңв”Җв”Җ evaluators/      # L1/L2/L3
в”ңв”Җв”Җ metrics/         # Thresholds
в””в”Җв”Җ reports/         # Results
```
